Interface IStudioWorkflowForEachStatement

For Each statement node in the workflow (but like Array.filter().map()).

IStudioWorkflowForEachStatement is a node entity that embodies a For Each loop in the IStudioWorkflow workflow program (actually Array.filter().map()), and it is a structure that executes a repeating task by composing a IStudioWorkflowBlockStatement block statement referred to as loop_block_id for the traversal target object expression.

And if a local variable is created in the loop statement, such as IStudioWorkflowCallStatement connector function call, they are all inserted into the array declared before the start of the loop block. However, if IStudioWorkflowBreakStatement break or IStudioWorkflowContinueStatement continue statements are used in the block, the local variables are not inserted into the array. In addition, if a loop is written double or triple, the array created by the loop is also inserted into the array of the upper loop.

For reference, the array containing the local variables created in the loop can be referred to as the IAstWorkflowLoopStatementResultIdentifier expression. And the individual elements of the loop target expression can be designated as the IAstWorkflowLoopStatementElementIdentifier expression. In addition, nodes placed within the loop block cannot be interconnected with nodes outside the loop block. This applies to all block statements.

In addition, you can write title and description to variables created through IStudioWorkflowForEachStatement. For more information, see IStudioWorkflowNodeDescriptable.

const output = [];
for (const elem of expression) {
  ...(statements of loop_block_id)
  output.push({ ..variables generated in the block });
}
...(statements of nexts[].node_id)

loop_block_id

loop_block_id: string

IStudioWorkflowBlockStatement.id of the block statement to be repeatedly executed.

A block is formed that groups the target node statements to be repeatedly executed, and the nodes in this block cannot be interconnected with nodes outside the block.

And if there is a node that creates a local variable, such as IStudioWorkflowCallStatement connector function call among the nodes in the block, all the created local variables are inserted into the array outside the block. However, if IStudioWorkflowBreakStatement break or IStudioWorkflowContinueStatement continue statements are used in the block, the local variables are not inserted into the array.

nexts

nexts: IStudioWorkflowNodeNext[]

List information of the next node statement.

After the execution of the current node statement is completed, a list of IStudioWorkflowStatement.id and a IStudioWorkflowNodeNext structure that describes the node statements to be executed next.

Note that the node that can be referred to must belong to a block at the same level as the current node. If the current node refers to a node belonging to a block such as for loop or IStudioWorkflowBranchStatement if statement, or a node belonging to a block such as a for loop or if statement refers to a node outside the block it belongs to, a validation error occurs by the workflow compiler.
