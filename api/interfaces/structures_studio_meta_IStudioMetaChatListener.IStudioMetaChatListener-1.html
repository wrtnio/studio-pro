Interface IStudioMetaChatListener

A set of functions that the Meta LLM client provides to the server.

IStudioMetaChatListener is an interface that defines a set of functions that the client provides to the Meta LLM server. Through this interface, the Meta LLM server RPCs (Remote Procedure Calls) the client's IStudioMetaChatListener functions.

  • completeWorkflow(workflow, related): Promise<void>

  • Workflow creation/modification is complete.

    If the Meta LLM server calls the client's prepareWorkflow function, and receives a non-null IStudioMetaChatWorkflowConfirm object as its return value, the Meta LLM server will actually create or modify the workflow.

explainFunctionCall

  • explainFunctionCall(plan): Promise<boolean>

  • Describes the function call plan.

    The Meta LLM server explains the function call plan to the user.

    The provided function call plan lists the functions that LLM wants to execute in order, and includes a description of the overall plan and the purpose of each function call.

    The client can read it and decide whether to perform the LLM function call execution or not. If it returns true, Meta LLM will call the functions as planned, and if it returns false, it will cancel the plan.

fillArgument

  • fillArgument(props): Promise<IFillArgumentResult>

  • Filling in arguments for a function call.

    In LLM Function Calls, there are cases where a person must fill in some argument values directly. For example, entering an authentication key for an account of another service such as Google, or directly uploading a file.

    IStudioMetaFunctionCall.arguments is a function called when a person must directly configure some (or all) of the argument values to be used in a function call.

    The client developer can develop a function that receives values from the user based on IOpenAiSchema and returns them.

    Currently, the following cases are received separately from the client in our service.

    • IOpenAiSchema.IString["x-wrtn-secret-key"]
    • IOpenAiSchema.IString.contentMediaType

Optionalsigterm

  • sigterm(): Promise<void>

  • A SIGTERM signal occurs on the backend server.

    A SIGTERM signal may occur on the backend server due to an update deployment or a random restart by Carpenter.

    And IStudioMetaChatListener.sigterm() is an RPC function called from the backend server to notify the client of the SIGTERM event.

    After receiving this function, the client should recognize that the backend server will be terminated within approximately 15 seconds and take corresponding measures. Of course, if there is already a response to the WebSocketConnector.join function, there is no need for the front end to do anything.
